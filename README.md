## Travel Planner Software

### Project Overview

This project aims to develop a software product that supports travelers in planning visits to cities (e.g., Poznań), based on user preferences, duration of stay, and interests. 

The system will generate a personalized travel plan including interesting places such as museums, parks, monuments, and other attractions.

## Communication Strategy

### Communication Channels

* GitHub: Main platform for task management, Product Backlog, and Sprint Backlog (using Issues) 
* Messaging Tool (e.g., WhatsApp / Discord): Quick communication and daily coordination
* Online Meetings (e.g., Google Meet / MS Teams): Sprint events and weekly meetings

---

### Meetings

#### Daily Communication

* Informal daily check-ins via messaging platform
* Focus on quick updates and coordination

#### Weekly Status Meeting

* The team will meet regularly to discuss progress: 

  * What has been done
  * What will be done next
  * Are there any problems or blockers

---

### Scrum Events

* Sprint Planning

  * Define Sprint goals and select tasks from the Product Backlog

* Sprint Review

  * Present completed work and gather feedback

* Sprint Retrospective

  * Reflect on team performance and identify improvements

---

### Task Management

* All tasks will be managed using GitHub Issues
* Each task will include:

  * Description
  * Acceptance criteria
  * Assigned team member
* Issues will be organized into Product Backlog and Sprint Backlog 

---

### Collaboration Rules

* All team members must regularly update task status
* Code changes must be shared through the repository
* Clear and respectful communication is required at all times
* Blockers must be communicated immediately to the Scrum Master
